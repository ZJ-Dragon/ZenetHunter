# /frontend/shell — Initial SPA Shell & Authentication

## Summary
This branch establishes the foundation for the ZenetHunter Frontend Single Page Application (SPA). It implements the core routing architecture, authentication state management (JWT), and the main application shell (Sidebar/Layout). It uses React Router v7, Tailwind CSS for styling, and Axios for API interaction.

## Changes

- **Dependencies**: Added `react-router-dom`, `axios`, `tailwindcss`, `lucide-react`, `clsx`, `tailwind-merge`.
- **Configuration**:
  - Configured Tailwind CSS with custom brand colors.
  - Configured `axios` with interceptors for automatic Token injection and 401 handling.
- **Architecture**:
  - **Router**: Implemented `createBrowserRouter` with protected routes (`RequireAuth`).
  - **AuthContext**: Created a React Context to manage User Token (persisted in `localStorage`).
- **Components**:
  - `AppShell`: Responsive Sidebar layout with navigation.
  - `Login`: Authentication page with integration to backend `/api/auth/login`.
  - `NotFound`: 404 Error page.
  - `RequireAuth`: Higher-order component for route protection.

## Key Parameters & Configuration

- **API URL**: Defaults to `http://localhost:8000/api` (configurable via `VITE_API_URL` env var).
- **Brand Colors**:
  - Primary: Sky Blue (`#0ea5e9` / `brand-500`).
  - Dark: `#0c4a6e` (Sidebar/Header).

## Local / CI Flow

1.  **Install Dependencies**:
    ```bash
    cd frontend
    npm install
    ```
2.  **Run Development Server**:
    ```bash
    npm run dev
    ```
3.  **Type Check**:
    ```bash
    npm run typecheck
    ```

## Acceptance Criteria

- ✅ User can navigate to `/login`.
- ✅ Valid credentials (admin/zenethunter) obtain a token and redirect to Dashboard.
- ✅ Invalid credentials show an error message.
- ✅ Accessing protected routes (`/`, `/devices`) without a token redirects to `/login`.
- ✅ Sidebar navigation works and highlights current route.
- ✅ 404 Page appears for unknown routes.
