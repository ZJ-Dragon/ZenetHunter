# /frontend/actions — Scan & Attack Controls

## Summary
This branch implements the interactive control mechanisms for ZenetHunter. It adds the ability to **Start Scan** from the dashboard and **Start/Stop Interference** (Attack) on specific devices. These actions are integrated into the UI with loading states and toast notifications for feedback.

## Changes

- **Dependencies**: Added `react-hot-toast` for notification feedback.
- **Services**:
  - `scanService`: Start scan (`POST /api/scan/start`).
  - `attackService`: Start/Stop attack (`POST /api/attack/start/:mac`, `POST /api/attack/stop/:mac`).
- **Components**:
  - `ScanButton`: A button component that triggers a network scan, handles loading state, and displays toasts.
  - `AttackControl`: A button component that toggles attack status for a device, handling the API call and visual feedback (Red for stop, Standard for start).
  - `ToastProvider`: Configuration for the toast notification system.
- **Integration**:
  - **Device List**: Added an "Actions" column with `AttackControl` for each device.
  - **Topology Page**: Added `ScanButton` to the header and `AttackControl` to the Node Drawer.
  - **Main Entry**: Integrated `ToastProvider` in `main.tsx`.

## Key Features
- **Visual Feedback**: Buttons show spinners while requests are processing.
- **Safety Checks**: Attack buttons are hidden for "Router" type devices to prevent accidental gateway denial.
- **Notifications**: Success/Error toasts appear for all actions.

## Acceptance Criteria
- ✅ "Start Scan" button appears on Topology page.
- ✅ Clicking "Start Scan" shows a loading spinner and then a success toast.
- ✅ "Interfere" button appears in Device List and Topology Drawer.
- ✅ Clicking "Interfere" changes button to "Stop Attack" (red) after success toast.
- ✅ Router/Gateway devices do not show attack options.

## Local / CI Flow
1.  **Backend**: Ensure backend is running (`uvicorn app.main:app`).
2.  **Frontend**: `npm run dev`.
3.  **Test Scan**: Click "Start Scan" -> Verify Toast -> Check Backend logs for scan start.
4.  **Test Attack**: Click "Interfere" on a dummy device -> Verify Toast -> Button turns Red.
