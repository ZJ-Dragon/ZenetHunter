# /frontend/devices — Device Management & Listing

## Summary
This branch implements the **Device List** page, allowing users to view all network devices discovered by the scanner. It includes a responsive table layout with real-time search (by name, IP, MAC, or vendor) and status filtering (Online, Offline, Blocked).

## Changes

- **Types**: Defined `Device`, `DeviceType`, `DeviceStatus`, `AttackStatus` interfaces in `frontend/src/types/device.ts`.
- **Service**: Created `deviceService` in `frontend/src/lib/services/device.ts` to handle API calls (`GET /devices`, `PUT /devices/:mac`, etc.).
- **UI Components**:
  - `DeviceList.tsx`: Main page component with state management for devices, loading, search, and filters.
  - **StatusBadge**: Visual indicator for device status (Green/Gray/Red).
  - **DeviceIcon**: Dynamic icon rendering based on device type (Router, PC, Mobile, IoT).
- **Routing**: Registered `/devices` route in `router.tsx`.

## Key Features
- **Search**: Filters devices instantly by Name, IP, MAC Address, or Vendor string.
- **Filter**: Dropdown to show only Online, Offline, or Blocked devices.
- **Refresh**: Manual refresh button to fetch latest data from backend.
- **Responsive**: Horizontal scroll support for tables on mobile.

## Acceptance Criteria
- ✅ Navigate to `/devices` shows a table of devices.
- ✅ Searching for "192.168" filters the list correctly.
- ✅ Selecting "Online" in filter dropdown shows only online devices.
- ✅ Device Types display appropriate icons (e.g., Smartphone icon for "mobile").
- ✅ Status column uses color-coded badges.

## Local / CI Flow
1.  **Backend**: Ensure backend is running (`uvicorn app.main:app`).
2.  **Frontend**: `npm run dev`.
3.  **Test**: Login -> Click "Devices" in sidebar -> Verify list loads.
