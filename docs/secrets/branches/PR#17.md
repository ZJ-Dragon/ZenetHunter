# /frontend/oobe — Out-of-Box Experience (Setup Wizard)

## Summary
This branch implements the **First-Time Setup Wizard** (OOBE) for ZenetHunter. It forces new installations to configure critical security and network settings before accessing the dashboard. The wizard covers Admin Password setup, Network Configuration, and Default Security Policy.

## Changes

- **Services & Types**:
  - `configService`: API client for `/api/config/status` and `/api/config/setup`.
  - `OOBEConfig` / `OOBESetupRequest` interfaces.
- **Components**:
  - `WizardLayout`: A reusable multi-step wizard layout with progress indicators.
  - `OOBEGuard`: A route guard that redirects users to `/setup` if the system is not configured.
- **Pages**:
  - `SetupWizard`: The main setup page implementing 3 steps:
    1.  **Admin Security**: Set admin password.
    2.  **Network Scan**: Configure target subnets and scan interval.
    3.  **Default Policy**: Choose between "Monitor Only" and "Auto-Block".
- **Routing**:
  - Added `/setup` route.
  - Wrapped protected routes with `OOBEGuard`.

## Key Features
- **Mandatory Setup**: Users cannot bypass setup if the backend reports `is_configured: false`.
- **Step-by-Step Guide**: Clear, segmented configuration process.
- **Security First**: Forces password reset/creation on first run.

## Acceptance Criteria
- ✅ Accessing root `/` redirects to `/setup` if unconfigured.
- ✅ Setup Wizard displays 3 steps.
- ✅ "Next" button is disabled until current step is valid (basic validation).
- ✅ Completing setup sends POST to `/api/config/setup` and redirects to `/login`.
- ✅ `OOBEGuard` allows access to dashboard if backend reports `is_configured: true`.

## Local / CI Flow
1.  **Backend**: Ensure backend is running.
2.  **Frontend**: `npm run dev`.
3.  **Test**:
    - If backend is fresh, visiting `http://localhost:5173` should redirect to `/setup`.
    - Complete the wizard.
    - Verify redirection to Login page.
