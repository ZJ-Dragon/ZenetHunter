# /frontend/topology — Network Topology Visualization

## Summary
This branch implements the **Topology Visualization** module, allowing users to view the network structure as a force-directed graph. It integrates real-time WebSocket updates to refresh the graph when devices are added or status changes. It also includes a "Node Drawer" that displays detailed device information when a node is clicked.

## Changes

- **WebSocket Infrastructure**:
  - `WebSocketContext.tsx`: Handles global WebSocket connection, authentication, and auto-reconnect.
  - `websocket.ts`: Defines standard event types (`deviceAdded`, `scanCompleted`, etc.).
  - `useWebSocketEvent` hook: Simplified subscription to specific backend events.
  
- **Topology Components**:
  - `TopologyGraph.tsx`: Wrapper around `react-force-graph-2d`, handling node painting and responsiveness.
  - `NodeDrawer.tsx`: Slide-out panel showing device details (IP, MAC, Status, Activity) and action buttons.
  - `Topology.tsx`: Main page component managing data fetching and WebSocket subscriptions.

- **Services & Types**:
  - `topologyService`: API client for `/api/topology`.
  - `TopologyNode` / `TopologyLink` interfaces.

## Key Features
- **Force-Directed Graph**: Visualizes devices and their relationships (currently star topology from Gateway).
- **Real-time Updates**: Automatically refreshes the graph when the backend emits `deviceAdded` or `deviceStatusChanged`.
- **Interactive Details**: Clicking a node opens a drawer with full device metadata.
- **Status Visualization**: Nodes are color-coded by status (Green=Online, Gray=Offline, Red=Blocked, Purple=Router).

## Acceptance Criteria
- ✅ Navigate to `/topology` renders the graph.
- ✅ Clicking a node opens the detail drawer.
- ✅ Drawer displays correct IP, MAC, and Status.
- ✅ WebSocket connection is established on login (check Console).
- ✅ Emitting a test event from backend triggers a graph refresh.

## Local / CI Flow
1.  **Backend**: Ensure backend is running (`uvicorn app.main:app`).
2.  **Frontend**: `npm run dev`.
3.  **Test**:
    - Check Topology page.
    - Open a second terminal and use `curl` to trigger a log or add a dummy device to backend to see if frontend reacts (console log "WS: Device Added").
