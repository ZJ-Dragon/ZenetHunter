# /production — OOBE hardening, limited admin scope, full-UI i18n sweep

## Summary
From 21053b81, this branch finalizes first-run safety, limits the default `admin/zenethunter` combo to settings-only access, and completes end-to-end internationalization (en + zh-CN/ja-JP/ko-KR/ru-RU) across core pages. UI surfaces (login, setup, dashboard/topology/devices/attacks/logs/settings) now align on WinUI-styled components, manual device overrides propagate everywhere, and replay/reset flows force users back to OOBE.

## Changes
### Authentication & roles
- `admin` with password `zenethunter` is now a limited “debug” account: can only open `Settings`; navigation elsewhere shows an i18n “Access denied” toast and reroutes back.
- Login no longer pre-fills admin or password; reset button removed to avoid accidental reuse.
- Auth context tracks a `limited_admin` flag in localStorage; logout clears it.

### Replay/reset safety
- Settings “Replay/Reset” now logs out, clears tokens/flags, and redirects to `/setup`, preventing stale admin reuse after a reset.

### UI/i18n unification
- Full i18n coverage (en default + zh-CN/ja-JP/ko-KR/ru-RU) for Dashboard, Devices, Topology, Logs, Attacks, Settings, Login, NotFound, Setup, etc.
- Attack dashboard and device lists use unified display fields (manual/display name/vendor) and listen to WS device updates to reflect manual overrides immediately.
- Setup wizard restyled with WinUI-like cards, input styling consistent with device search fields, and dark-mode-friendly theming.
- NotFound/Dashboard/Topology/Logs/Devices/Attack pages align typography, spacing, and translated labels.

## Interfaces & events
- Frontend listens to WS events (`deviceAdded`, `deviceUpdated`, `deviceStatusChanged`, `deviceRecognitionUpdated`) in attack views to keep device data and manual labels in sync.
- No backend endpoint changes in this slice; frontend replay still invokes the existing `/replay` endpoint and then forces OOBE.

## Config & environment
- API base: `VITE_API_URL` (default `http://localhost:8000/api`).
- Limited admin behavior is client-side only; tokens for that login are marked with `limited_admin` in local storage to gate navigation.

## Notes for release
- After hitting “Replay/Reset”, users are redirected to Setup and must recreate admin credentials.
- Manual overrides now render consistently across all device lists (including attack view) and survive websocket refresh cycles.
